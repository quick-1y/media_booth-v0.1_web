# Parking Media Wall

Готовый каркас проекта для медиастендов парковки на **FastAPI + HTML/CSS/JS + Poetry + Docker + Nginx**.

Проект учитывает ваши текущие наработки: экран с секциями **часы работы**, **свободные места**, **тарифы**, **рекламная карусель**, скрытое открытие меню настроек, загрузку настроек из файла конфигурации и работу через **backend proxy** к JSP-парсеру парковки.

## Меню настроек
Меню открывается тремя способами:
1. почти невидимая кнопка в правом верхнем углу;
2. клавиши **Alt + Shift + S**;
3. многократный тап по заголовку на тач-экране.

Внутри меню есть вкладки:
- **Общие** — заголовки экрана, название локации;
- **Парковка** — URL парсера, интервалы, таймауты, TLS;
- **Контент** — часы работы, тарифы;
- **Реклама** — путь к папке, список ручных файлов, время карусели;
- **Оформление** — акцентные цвета;
- **Диагностика** — тест парсера, просмотр сырого YAML, путь к конфигу.

## Быстрый старт

```bash
cp .env.example .env
docker compose --env-file .env up -d --build
```

Откройте:

```text
http://<HOST>:8085/
```

## Почему backend proxy важен
В старом подходе страница сама ходила к `parking_free_public.jsp` из браузера. Это ломается при CORS, mixed content и когда парсер доступен только по внутреннему адресу. В этой версии браузер общается только с FastAPI, а уже он читает JSP по сети.
